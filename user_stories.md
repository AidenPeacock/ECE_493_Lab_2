# CMS User Stories (Extracted from SRS)

Source SRS: Conference Management System (CMS) SRS v1.0. :contentReference[oaicite:2]{index=2}

## US-01 Register for an account
As a prospective CMS user (author/reviewer/editor/attendee), I want to register with a unique, valid email and a secure password so that I can access authorized CMS features.  
(From SRS Feature 3.1 / FR1)

## US-02 Log in
As a registered user, I want to log in with my username and password so that I can access CMS features appropriate to my role.  
(From SRS Feature 3.2 / FR2)

## US-03 Change password
As a registered user, I want to change my password so that I can maintain account security.  
(From SRS Feature 3.2 / FR3)

## US-04 Submit a paper manuscript
As an author, I want to submit a manuscript and its metadata so that it can enter the peer review process.  
(From SRS Feature 3.3 / FR4)

## US-05 Save a paper submission in progress
As an author, I want to save a submission draft at any stage so that I can complete it later without losing work.  
(From SRS Feature 3.3 / FR5)

## US-06 Assign referees to a submitted paper
As an editor, I want to assign three referees to each submitted paper (using their email addresses) so that the paper can be reviewed by appropriate experts.  
(From SRS Feature 3.4 / FR6)

## US-07 Referee accepts an invitation and gets the paper in their account
As a referee, I want to accept a review invitation so that the paper appears in my account as an assigned review task.  
(From SRS Feature 3.4 / FR7)

## US-08 Review an assigned paper
As a referee, I want to fill out and submit a review form for an assigned paper so that the editor receives my evaluation.  
(From SRS Feature 3.5 / FR8)

## US-09 Editor makes a final decision after three reviews
As an editor, I want to record an accept/reject decision after all three reviews are submitted so that the author is notified and the system reflects the outcome.  
(From SRS Feature 3.5 / FR9)

## US-10 Generate a conference schedule for accepted papers
As an administrator/editor, I want the system to generate a schedule (times/rooms) for accepted papers so that authors and attendees can plan participation.  
(From SRS Feature 3.6 / FR10)

## US-11 Edit and publish the conference schedule
As an editor, I want to edit and update the generated schedule so that the final published schedule is correct.  
(From SRS Feature 3.6 / FR11)

## US-12 Pay for conference attendance and receive a ticket
As an authorized user/attendee, I want to pay (by credit card) and receive a payment confirmation ticket so that I can attend the conference.  
(From SRS Feature 3.7 / FR12)

---

## Notes on SRS statements that imply additional user stories (not explicit FRs)
The SRS mentions **public announcements** and that the **price list is readable for guest users**, and that the **final schedule is announced on the CMS webpage**. Those are not specified as functional requirements with FR IDs in the provided SRS text, but they are implied system capabilities.
